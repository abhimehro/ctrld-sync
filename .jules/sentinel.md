## 2025-03-03 - Denial of Service (DoS) via URL Parsing Resource Exhaustion
**Vulnerability:** The application parsed and validated remote blocklist URLs using `httpx.URL()` and DNS resolution without first enforcing a strict string length limit. An attacker providing an exceptionally long URL (e.g., millions of characters) via config or CLI could cause a localized Denial of Service (DoS) due to CPU/memory exhaustion during the parsing or DNS lookup phase (e.g., triggering `idna` codec `label too long` errors after substantial processing time).
**Learning:** Network operations and complex string parsing (like URLs or regex) are computationally expensive and vulnerable to resource exhaustion. Even if the parsing library eventually throws an error (like `idna`), the CPU and memory cost is already incurred.
**Prevention:** Enforce strict maximum length limits (e.g., `MAX_URL_LENGTH = 2048`) on user-provided strings *before* performing computationally expensive operations, parsing, or network requests.
